Q : 그렇다면 http 1,2 차이 ?

A : 요청 보내고 응답 받는 걸 한단계 한단계 함 , 2.0 요청은 주고받는 걸 동시에

Q : http와 https 차이 ?

A : http는 보안이 되지 않는 단순 평문

https는 http에서 tls 보안이 추가된 것

최근에는 http를 웹 브라우저에서 지원하지 않는다

https로 대부분 처리함

---

Q : http2의 특징?

A :  2 요청 - 응답 순차적인 것이 아니라 보내고 받는 것을 동시에

1 에서 발전된 것

Q : 쿠키나 세션 언제 사용?

A :  쿠키는 클라이언트 쪽에서 세션 id를 보관해서 서버쪽의 부하를 줄임

세션은 서버쪽에서 관리가 좀 더 필요

---

Q : 최근에 한 프로젝트에서는 로그인 상태?

A :  클라이언트 → 서버

서버에서 토큰 내려주면 로컬 스토리지에서 관리

탭 별로 영구적으로 가지고 있고

Q : http method에 대해?

A : get post put patch

get 는 db를 조회

post는 생성하거나 요청처리

put은 전체적인 수정, 전체적인 값

patch 는 부분적인 일부분을 수정하는 메소드

---

Q : 멱등성이 보장되지 않는 메소드?

A : 멱등성에 대한 설명은 어려움

get과 put은 멱등성 보장, post patch는 보장 x

---

Q : put과 patch의 차이점 ?

A : 전체적인 데이터를 수정해서 전체 데이터를 덮는 기능

patch는 일부분 데이터만 수정하는 것

---

A : rest는 음.. 메소드와 서버에서 api를 짤 때 지키면 좋은 규칙

찾아보지 못함

restful은 rest가 잘 지켜진 api
