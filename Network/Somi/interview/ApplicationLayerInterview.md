# 응용계층 모의면접

Q. HTTP 특징

A.
무상태성, 비연결성을 말씀드릴 수 있어요.
무상태성은 서버가 클라이언트를 상태 저장하지 않아요.
따라서 클라가 만약 페이지에 로그인을 했다가 다음 페이지로 이동하면 클라의 상태를 모르기때문에 재로그인 상황 발생. 계속 요청해야 한다는 단점이 있습니다.
서버를 확장시킬때의 장점이 있다는 것을 알 수 있습니다.
비연결성은 서버와의 연결 끊으면 네트워크 연결자체를 끊어집니다.
서버가 많은 클라와의 연결을 유지했ㅇ르 때의 리소스 줄일 수 있어 많은 클라 연결 가능하다는 장점이 있습니다.

Q. preflight 요청이란
A.
어떤 해당 리소스를 요청으로 인해 서버의 데이터가 변경되는 delete같은 경우 preflight를 보내 이 요청이 안전한지 먼저 확인
조금 더 안정적으로 요청이 이루어질 수 있도록 하는 방식

Q. 세션 기반 토큰 기반 인증
A.
세션 기반인증)
서버가 세션id를 사용자에게 보내주고 사용자는 쿠키에 저장해두었다가 서버가 세션아이디를 받으면 그것으로 인증하는 방식
토큰 기반인증)
서버가 토큰을 jwt 토큰을 발급함녀 사용자는 토큰을 가지고 있다가 요청보냄
서버는 해당 토큰 유효한지 판단
