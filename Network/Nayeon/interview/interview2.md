## 서기

1. 전송계층에 대해 설명
    1. 전송계층은 데이터를 주고받을때 어떻게 주고받을지를 전적으로 담당
    2. TCP, UDP
    3. TCP는 보안적으로 좀 더 우수
        1. 정확히 데이터를 받았는지 인증하는 데이터를 추가적으로 보내줌
        2. 그러나 성능적으로 떨어짐
    4. UDP
        1. 서로가 주고받았는지 확인 불가
        2. 속도적으로 우월해서 활용
2. TCP가 신뢰면에서 우수 어떤방식?
    1. 정확히말씀드릴 수 없지만
    2. 3way handshake
        1. 보내는쪽에서 전송하면 ack을 추가적인 플래그를 보내서 
        2. 아 내가 이걸 지금 받았다 확인하는 것을 통해
        3. 신뢰를 지키는 것으로 알고 있음
3. 3 way, 4 way 단계 수 차이가 나는 이유
    1. 죄송합니다. 면접 후 추가적으로 보완하겠습니다.
4. 소켓통신과 http 통신의 차이점
    1. http 통신은 프로토콜 통신으로 알고 있습니다.
    2. 도메인을 통해서 api에 데이터를 담아서 전송하는 것으로 알고이ㅣㅆ고
    3. 소켓은 좀 더 하드웨어적으로 알고 있음
5. 하나의 서버에서 여러앱이 동시에 실행. 포트 충돌 가능 해결방법
    1. 크게 2가지
        1. 서버 증설하는 방식
        2. 서버의 성능을 향상시키는 방식 → 속도를 더 빠르게 → 데이터 더 많이
6. TCP에서 데이터가 여러개로 나눠져서 전송될때 순서를 어떻게 보장하는지 아시나요?
    1. 죄송합니다. 알지 못합니다.
7. 네트워크 계층에 대해서 대략적 설명
    1. 네트워크는 ..
    2. 죄송합니다.
8. IP주소와 포트의 차이를 아실까요?
    1. IP주소는 크게 일단
        1. 도메인으로 변환
            1. IPv6와 8을 알고 있음
                1. 도메인 확장을 위해 8을 사용하는 것으로 알고 있음
    2. 포트
        1. 클라우드 상에서 다른 아키텍처를 연결하기위해 사용하는 것으로
9. 공인 IP와 사설 IP의 차이
    1. 공인 IP는 퍼블릭 IP
        1.  외부에서 출처로부터 모두 허용
    2. 프라이빗 IP
        1. 내부에서 데이터를 지킬 때 사용되는 IP라고 알고 있음
10. NAT에 대해서 알고 계시나요?
    1. 클라우드 상에서 아키텍처를 연결할 때 데이터가 연결되는 과정에서 보안을 지키기 위해 사용
    2. 중간에 통로망을 뚫어서 그 속으로만 데이터 전송. 외부적으로 누설되지ㅣ 않도록 통로를 뚫는 작업
11. 라우팅 알고리즘에 대해 알고 있나요?
    1. 잘 알지 못합니다. 죄송합니다.
12. 라우터와 스위치의 차이
    1. 죄송합니다.
